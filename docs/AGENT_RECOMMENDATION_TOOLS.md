# Agent Recommendation Tools (Copy Prompt / Dislike / Archive)

## 🟩 USER NOTES
- Нужны инструменты рекомендаций прямо в интерфейсе:
  - кнопка **«Скопировать промпт»** для мгновенного запроса в Codex/VS Code;
  - кнопка **«Дизлайк»**, чтобы идея уходила в архив как нерелевантная;
  - семантика обучения на реализованных и отклонённых идеях, чтобы советы становились точнее.

## 🟦 ASSISTANT NOTES

## Цель
Сделать рекомендации не «просто карточками», а **операционным инструментом действий**:
1. увидел идею;
2. нажал «Скопировать промпт» и запустил реализацию;
3. или нажал «Дизлайк» и система учла это в следующих советах.

## Базовые сущности
### 1) Recommendation Card
Поля карточки:
- `id`
- `title`
- `context` (почему эта рекомендация сейчас)
- `prompt_for_agent` (готовый текст для кнопки «Скопировать промпт»)
- `impact` (ожидаемый эффект)
- `effort` (S/M/L)
- `tags` (например: kinderly, heys, health, operations)
- `status` (`new`, `copied`, `implemented`, `disliked`, `archived`)

### 2) Feedback Event
События на карточке:
- `copied_prompt`
- `implemented`
- `disliked`
- `archived`

### 3) Learning Profile (простая семантика)
- `positive_tags`: теги идей, которые чаще копируют/реализуют;
- `negative_tags`: теги идей, которые чаще дизлайкают;
- `preferred_format`: какие форматы советов заходят (короткие, с чеклистом, с KPI).

## UX-поток
1. Карточка показывается в ленте рекомендаций.
2. Пользователь нажимает:
   - **«Скопировать промпт»** → карточка помечается `copied`.
   - **«Дизлайк»** → карточка помечается `disliked` и уходит в архив.
3. Раз в день/неделю агент пересчитывает релевантность карточек по истории событий.

## Правила адаптации рекомендаций (v1)
- Если тег часто в `implemented` → повышаем приоритет похожих рекомендаций.
- Если тег часто в `disliked` → понижаем частоту показа.
- Если у карточек с высоким `effort` частые дизлайки → предлагать более короткие шаги.
- Если копируют, но не реализуют → добавлять более конкретный «первый шаг за 15 минут».

## Минимальный формат кнопок на карточке
- Primary: `Скопировать промпт`
- Secondary: `Дизлайк`
- Tertiary: `Уже реализовано`

```text
3) [шаг 3]

Требования:
- сохранить разделение 🟩 USER NOTES / 🟦 ASSISTANT NOTES,
- обновить связанные docs,

```

1. Добавить хранение карточек и событий (файлово/JSON достаточно для старта).
2. Реализовать действия `copy / dislike / implemented`.
3. Добавить nightly-скрипт ранжирования рекомендаций по истории фидбэка.
