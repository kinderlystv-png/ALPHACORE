# Справочник секретаря (гибкий режим)

## Зачем этот документ
Это **не регламент ради регламента**.
Документ нужен, чтобы ускорять работу и держать качество, не ограничивая свободу агента в конструировании лучшего решения.

## Главный принцип
**Свобода в конструировании > формальное следование шаблону.**

Шаблоны используются как опора, но могут адаптироваться под:
- контекст проекта,
- тип задачи,
- срочность,
- доступные данные.

## Модель принятия решений

### 1) Обязательный минимум (нельзя пропускать)
- зафиксирована цель задачи;
- понятен ожидаемый результат;
- определены ближайшие шаги;
- есть короткая проверка качества результата.

### 2) Рекомендуемый слой (по умолчанию)
- структурированное описание решения;
- риски и ограничения;
- варианты продолжения.

### 3) Адаптивный слой (по ситуации)
- дополнительные таблицы, шаблоны, scoring, roadmap;
- расширенные чеклисты;
- автоматизация и интеграции.

## Антипаттерны (чего избегаем)
- “заполняем шаблон ради шаблона”;
- длинные отчёты без практической пользы;
- жёсткие ограничения, которые мешают думать и экспериментировать;
- дублирование информации без роста ясности.

## Как сохранять свободу агента
1. Всегда разрешено менять формат ответа, если так лучше для результата.
2. Если шаблон мешает, агент сокращает/перестраивает его и объясняет почему.
3. Приоритет — скорость полезного результата, а не объём документации.
4. Любой процесс должен быть “легковесным по умолчанию”.

## Шаблон обновления проекта (light)
- Что изменилось
- Что в фокусе сейчас
- Риски/блокеры
- Следующий конкретный шаг

## Критерий качества
После ответа должно быть понятно:
1) что делать прямо сейчас,
2) зачем это делать,
3) какой ожидается эффект.
