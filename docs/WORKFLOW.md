# ALPHACORE — Engineering Workflow

## Branching
- `main` — стабильная ветка
- feature-ветки: `feat/<scope>-<short-name>`
- bugfix-ветки: `fix/<scope>-<short-name>`

## Commit convention
- `feat:` новая функциональность
- `fix:` исправления
- `chore:` техдолг/инфраструктура
- `docs:` документация

## Pull Request DoD
- [ ] Есть описание мотивации и изменений
- [ ] Есть план отката (если рискованно)
- [ ] Добавлены/обновлены тесты
- [ ] Прошли CI проверки
- [ ] Обновлена документация (если нужно)

## Гибкость процесса (важно)
1. Workflow — это ориентир, а не жёсткая клетка.
2. Можно сокращать артефакты, если задача маленькая и риск низкий.
3. Можно расширять процесс, если задача критичная или высокорисковая.
4. Приоритет: скорость ценного результата + ясность следующего шага.
