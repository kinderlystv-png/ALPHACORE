# Recommendation UI Spec (Mobile-first)

## 🟩 USER NOTES
- Хочу прямо в интерфейсе быстро решать судьбу идеи:
  - годная → кнопка «Скопировать промпт»;
  - не зашла → «Дизлайк» и в архив.

## 🟦 ASSISTANT NOTES

## Цель интерфейса
Снизить трение между "увидел идею" и "запустил реализацию" до одного клика.

## Карточка рекомендации
Обязательные блоки:
1. **Заголовок** (что делать).
2. **Почему сейчас** (контекст).
3. **Готовый промпт** (скрытый/раскрываемый блок).
4. **Кнопки действий**:
   - Primary: `Скопировать промпт`
   - Secondary: `Дизлайк`
   - Ghost: `Уже реализовано`

## Состояния карточки
- `new`
- `copied`
- `implemented`
- `disliked`
- `archived`

## UX-сценарии
### A. Copy Prompt
1. Пользователь нажимает `Скопировать промпт`.
2. Текст уходит в буфер обмена.
3. Показывается тост: "Промпт скопирован".
4. Карточка получает статус `copied`.

### B. Dislike
1. Пользователь нажимает `Дизлайк`.
2. Карточка скрывается из основной ленты.
3. Записывается событие `disliked` и карточка уходит в архив.

### C. Implemented
1. Пользователь нажимает `Уже реализовано`.
2. Записывается событие `implemented`.
3. Карточка скрывается из активных рекомендаций.

## Архив
В архиве доступны фильтры:
- по действию (`disliked`, `implemented`),
- по тегам (`kinderly`, `heys`, `health`),
- по дате.

## Пояснение по семантике
- Частые `disliked` по тегу → понижаем показ идей с этим тегом.
- Частые `implemented` → поднимаем похожие идеи.
- Частые `copied` без `implemented` → предлагать более короткие/конкретные шаги.

## MVP-метрики
1. CTR кнопки `Скопировать промпт`.
2. Доля `implemented` от `copied`.
3. Доля `disliked` по тегам.
4. Time-to-action (от показа карточки до действия).
